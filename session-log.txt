When PM2 kills Jin without clean WebSocket close frame, stale connections persist.

Fixes implemented this session:

  2. Shutdown timeout guard — 8s force-exit before PM2 escalates to SIGKILL
  3. num_connections monitoring — intercepts raw WS hello message, logs count
  4. Event delivery watchdog — checks every 5min, force-reconnects after 30min silence

Emergency recovery (if Jin stops receiving messages):
  Option A: pm2 restart jin (should work now with proper graceful shutdown)
